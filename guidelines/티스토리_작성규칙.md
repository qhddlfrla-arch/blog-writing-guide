# 티스토리 블로그 작성 규칙

## 핵심 원칙: 마크다운 필수

티스토리는 **마크다운을 적극 활용**합니다.

## 필수 사용 마크다운

### 1. 제목 구조
```markdown
# 메인 제목 (1회만 사용)

## 주요 섹션 제목

### 소제목 (필요시에만 최소한으로 사용)
```

### 2. 강조
```markdown
**중요 내용 강조**
*이탤릭 강조*
```

### 3. 리스트
```markdown
- 항목 1
- 항목 2
  - 하위 항목

1. 순서 항목 1
2. 순서 항목 2
```

### 4. 테이블
```markdown
| 항목 | 설명 | 비고 |
|------|------|------|
| 내용1 | 설명1 | 비고1 |
| 내용2 | 설명2 | 비고2 |
```

## 출력 포맷

```markdown
# 메인 제목

후킹 도입부 (2-3문장)

## 섹션 1 제목

본문 내용 (6-8문장 이상)
- 구체적 정보
- 실제 데이터
- 사례 포함

## 섹션 2 제목

본문 내용 (6-8문장 이상)

### 필요시 소제목 (최소한으로)

상세 내용

## 마무리

핵심 요약 (3-5문단)

---

**태그**: 태그1, 태그2, 태그3, 태그4, 태그5

**썸네일 아이디어**:
1. 구체적인 썸네일 아이디어
2. 구체적인 썸네일 아이디어
```

## 콘텐츠 작성 원칙

### 1. 분량
- 기본: 2,500~3,500자
- 사용자 지정 시 해당 분량 준수

### 2. 제목 사용 규칙 (## 중심 구조)
- `#` 메인 제목: 1회만 사용
- `##` 주요 섹션: 3-5개 정도 (핵심 구조)
- `###` 소제목: **가급적 지양**, 꼭 필요한 경우에만 최소한으로 사용
- ## 중심의 명확한 구조를 유지하여 가독성 확보

### 3. 심층 작성 (충실한 분량)
- 각 섹션당 **반드시** 최소 6-8문장 이상 작성
- 피상적인 설명 배제, 심층적인 내용으로 구성
- 구체적인 수치, 통계, 데이터 필수 포함
- 실제 사례와 실용적인 팁 제공
- 표를 활용한 비교 분석
- 리스트로 명확한 정보 정리

### 4. 팩트 기반 작성 (허위 경험 절대 금지)
- 객관적 정보와 검증된 사실만 활용
- **절대 금지**: "제가 직접 써보니", "어제 다녀온", "직접 사용해보니", "제가 먹어본 결과" 등 가짜 체험담
- 허위 사용 후기 및 경험담 작성 절대 금지
- 출처가 명확한 데이터와 통계만 활용

### 5. 후킹 도입부
- 흥미로운 수치나 통계로 시작
- 독자의 궁금증을 유발하는 질문 활용
- 글을 읽어야 할 명확한 이유 제시

### 6. 가독성
- 한 문단: 3-5문장 구성
- 마크다운으로 시각적 구조화
- 표와 리스트 적극 활용
- 적절한 줄바꿈과 여백

### 7. 필수 포함 요소
- **핵심 요약(마무리)**: 본문의 주요 내용을 3-5문단으로 압축
- **썸네일 아이디어**: 구체적이고 실현 가능한 아이디어 2개
- **태그**: 쉼표로 구분 (태그1, 태그2, 태그3)

## 작성 체크리스트

- [ ] # 메인 제목을 1회만 사용했는가?
- [ ] ## 주요 섹션이 적절히 구분되었는가?
- [ ] ### 소제목을 남발하지 않았는가?
- [ ] **강조** 마크다운을 활용했는가?
- [ ] 표나 리스트를 적절히 사용했는가?
- [ ] 각 섹션이 6-8문장 이상인가?
- [ ] 가짜 체험담이나 허위 정보가 없는가?
- [ ] 후킹 도입부가 있는가?
- [ ] 문단별 3-5문장 구성이 지켜졌는가?
- [ ] 핵심 요약과 썸네일 아이디어가 포함되었는가?
- [ ] 태그가 쉼표로 구분되었는가?
- [ ] 분량이 2,500~3,500자인가?

## 주의사항

1. **마크다운 과용 금지**
   - ### 소제목 남발하지 않기
   - 불필요한 강조 자제
   - 자연스러운 흐름 유지

2. **절대 포함하지 말 것**
   - "이미지를 삽입하세요" 같은 메타 가이드
   - 의미 없는 문장 반복으로 분량 채우기 (늘어쓰기 금지)
   - SEO 키워드 나열식 작성

3. **품질 우선**
   - 분량 채우기보다 내용의 깊이
   - 실질적 가치 제공
   - 독자 중심 관점
   - 마크다운은 가독성 향상 도구
