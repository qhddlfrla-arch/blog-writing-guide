# 블로그 콘텐츠 작성 시스템 사용 가이드

## 프로젝트 개요

플랫폼별(네이버, 티스토리, 애드센스, 브랜드) 블로그 글 작성을 위한 체계적인 가이드라인과 템플릿을 제공합니다.

## 폴더 구조

```
blog/
├── README.md                           # 프로젝트 소개
├── USAGE_GUIDE.md                      # 이 파일
├── guidelines/                         # 플랫폼별 작성 규칙
│   ├── 네이버블로그_작성규칙.md
│   ├── 티스토리_작성규칙.md
│   ├── 애드센스_작성규칙.md
│   └── 브랜드블로그_작성규칙.md
├── templates/                          # 플랫폼별 템플릿
│   ├── 네이버블로그_템플릿.txt
│   ├── 티스토리_템플릿.md
│   ├── 애드센스_템플릿.md
│   └── 브랜드블로그_템플릿.md
└── examples/                           # 실제 작성 예제
    ├── 네이버_난방비절감_예제.txt
    ├── 티스토리_공기청정기_예제.md
    ├── 애드센스_홈오피스_예제.md
    └── 브랜드_쇼핑몰CS_예제.md
```

## 사용 방법

### 1단계: 플랫폼 선택
작성하려는 블로그 플랫폼을 결정합니다.
- **네이버 블로그**: 일반 블로거, 순수 텍스트 선호
- **티스토리**: 마크다운 지원, 개인 블로거
- **애드센스**: 수익형 블로그, SEO 중요
- **브랜드 블로그**: 기업/제품 공식 블로그

### 2단계: 가이드라인 확인
`guidelines/` 폴더에서 해당 플랫폼의 작성 규칙을 숙지합니다.

**핵심 차이점:**
| 플랫폼 | 마크다운 사용 | 태그 형식 | 분량 | 특징 |
|--------|---------------|-----------|------|------|
| 네이버 | ❌ 절대 금지 (#, ##, **, -, \|, ---, ``` 등) | #태그1 #태그2 | 2,500~3,500자 | PLAIN TEXT만 |
| 티스토리 | ✅ 필수 (## 중심, ### 지양) | 태그1, 태그2 | 2,500~3,500자 | 마크다운 적극 활용 |
| 애드센스 | ✅ 필수 (## 중심, ### 지양) | #태그1 #태그2 | 3,000~3,500자 | SEO 최적화 |
| 브랜드 | ✅ 필수 (## 중심, ### 지양) | 태그1, 태그2 | 2,500~3,500자 | 브랜드 톤앤매너 |

### 3단계: 템플릿 복사
`templates/` 폴더에서 해당 플랫폼 템플릿을 복사하여 새 파일로 작성합니다.

### 4단계: 예제 참고
`examples/` 폴더의 실제 작성 예제를 참고하여 구조와 스타일을 파악합니다.

### 5단계: 작성 및 검증
템플릿을 기반으로 글을 작성한 후, 각 플랫폼의 체크리스트로 검증합니다.

## 공통 작성 원칙

### ✅ 반드시 지켜야 할 것

1. **심층 작성 (충실한 분량)**: 각 섹션 **반드시** 6-8문장 이상
2. **피상적 설명 배제**: 심층적인 내용으로 구성, 추상적 표현 지양
3. **데이터 기반**: 구체적인 수치, 통계, 실용적인 팁 필수 포함
4. **팩트 기반**: 객관적 정보, 검증된 사실만 활용
5. **허위 경험 절대 금지**: "제가 직접 써보니", "어제 다녀온", "직접 사용해보니" 등 가짜 체험담 금지
6. **후킹 도입부**: 수치, 통계, 흥미로운 질문으로 시작
7. **가독성**: 한 문단 3-5문장
8. **필수 요소**: 핵심 요약(마무리) + 썸네일 아이디어 2개
9. **분량**: 기본 2,500~3,500자 (애드센스는 3,000자 이상)

### ❌ 절대 하지 말아야 할 것

1. 허위 사용 후기, 가짜 체험담 ("제가 직접 써보니", "어제 다녀온", "직접 먹어보니" 등)
2. 과장된 정보
3. 의미 없는 문장 반복 (늘어쓰기 금지)
4. 피상적인 설명으로 분량 채우기
5. "이미지를 삽입하세요" 같은 메타 가이드 포함
6. SEO 키워드 나열식 작성 (애드센스)
7. 플랫폼별 포맷 규칙 위반

## 플랫폼별 주의사항

### 네이버 블로그
⚠️ **마크다운 기호 절대 사용 금지**
- `#`, `##`, `**`, `*`, `-`, `|`, `---`, ``` 등 모든 마크다운 기호 제거
- [제목], [본문], [마무리], [태그], [썸네일 아이디어] 포맷 엄수
- 태그: `#태그1 #태그2` 형식

### 티스토리
✅ **마크다운 적극 활용**
- `#` 메인 제목 1회만
- `##` 주요 섹션 3-5개 (핵심 구조)
- `###` 소제목은 **가급적 지양** (남발 금지)
- 표, 리스트 활용
- 태그: `태그1, 태그2` 쉼표 구분

### 애드센스
✅ **마크다운 + SEO 최적화**
- `#` 메인 제목 1회, `##` 주요 섹션 3-5개
- `###` 소제목 **가급적 지양** (필요시에만 최소한으로)
- 키워드 자연스럽게 분산 (나열 금지)
- 메인 키워드: 제목 1회 + 본문 3-5회
- 검색 의도 파악 후 충실한 답변
- 3,000자 이상 권장
- 태그: `#태그1 #태그2`
- 허위 경험담 절대 금지

### 브랜드 블로그
✅ **마크다운 + 브랜드 아이덴티티**
- `#` 메인 제목 1회, `##` 주요 섹션 3-5개
- `###` 소제목 **가급적 지양** (필요시에만 최소한으로)
- 브랜드 톤앤매너 일관성
- 고객 중심 관점 (자화자찬 금지)
- 실제 고객 사례만 사용 (허가 필수)
- 허위 후기 절대 금지
- 자연스러운 Call-to-Action
- 태그: `태그1, 태그2`

## 품질 체크리스트

글 작성 완료 후 다음을 확인하세요:

- [ ] 플랫폼 포맷 규칙 준수 (네이버: 마크다운 없음 / 기타: 마크다운 있음)
- [ ] 각 섹션 6-8문장 이상
- [ ] 한 문단 3-5문장 구성
- [ ] 가짜 체험담, 허위 정보 없음
- [ ] 후킹 도입부 있음
- [ ] 팩트 기반 객관적 정보
- [ ] 핵심 요약(마무리) 포함
- [ ] 썸네일 아이디어 2개 포함
- [ ] 태그 형식 올바름 (플랫폼별 상이)
- [ ] 분량 충족 (2,500~3,500자, 애드센스 3,000자+)
- [ ] 불필요한 늘어쓰기, 메타 가이드 없음

## 자주 묻는 질문

**Q: 분량을 더 늘리고 싶은데 어떻게 하나요?**
A: 불필요한 늘어쓰기 대신 각 섹션의 깊이를 더하세요. 구체적인 데이터, 사례, 비교 분석을 추가하면 자연스럽게 분량이 늘어납니다.

**Q: 네이버 블로그에서 강조하고 싶은 부분은 어떻게 표현하나요?**
A: 마크다운을 사용할 수 없으므로 줄바꿈, 띄어쓰기, 문장 구조로 강조합니다. "중요한 점은 이것입니다" 같은 명시적 표현을 활용하세요.

**Q: 실제 사용 후기를 작성해도 되나요?**
A: 본인이 직접 사용한 제품이라면 가능하지만, 사용하지 않은 제품에 대해 "직접 사용해보니" 같은 거짓 체험담은 절대 금지입니다.

**Q: 애드센스 블로그에서 키워드를 어떻게 넣나요?**
A: 자연스럽게 문맥에 맞춰 사용하세요. 나열식 키워드 삽입은 검색 엔진과 독자 모두에게 좋지 않습니다.

**Q: 썸네일 아이디어는 왜 필요한가요?**
A: 블로그 글의 클릭률을 높이기 위한 시각적 요소입니다. 구체적이고 실현 가능한 아이디어를 제시하세요.
